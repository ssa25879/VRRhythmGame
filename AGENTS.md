# VRBeatSaber Agent Instructions

## 기본 응답 규칙

- 한국어로 답변한다.
- 정보를 제공할 때는 필요한 내용만 간결하고 정확하게 답한다.
- 변경 가능성이 있거나 불확실한 정보는 확인 후 답한다.
- 사용자가 명시적으로 멈추라고 하지 않는 한, 요청받은 작업은 가능한 범위에서 끝까지 진행한다.

## 프로젝트 범위

- 프로젝트 루트는 이 `AGENTS.md` 파일이 위치한 저장소 루트로 판단한다.
- 현재 작업 환경의 로컬 경로 예시는 `D:\work\VRBeatSaber`이지만, 다른 환경에서 클론하면 경로가 달라질 수 있으므로 지침과 작업은 프로젝트 루트 기준 상대 경로를 우선 사용한다.
- 이 저장소는 Unity 기반 VR 리듬 게임 프로토타입이다.
- 프로젝트 관련 문서와 작업 관리는 이 저장소 내부 파일 기준으로 유지한다.

## 문서 관리

- `AGENTS.md`: AI 작업 공통 지침을 관리한다.
- `Task.md`: 남은 작업, 완료 작업, 대기 항목을 관리한다.
- `Log.md`: 프로젝트 작업 이력과 확인 결과를 관리한다.
- `README.md`: GitHub에서 볼 프로젝트 간단 소개를 관리한다.
- `Log.md` / `Task.md` / `AGENTS.md`를 수정할 때는 별도 검토 요청 없이 수정 후, 어떤 파일을 수정했는지만 보고한다.
- `Log.md`, `Task.md`, `AGENTS.md`, 외부 작업일 로그 파일은 백업하지 않는다.

## 외부 작업 로그

- 현재 작업 환경에서는 작업일 로그를 `D:\Codex\Log` 아래에 프로젝트명 포함 파일명으로 관리한다.
- 다른 환경에서 해당 경로가 없으면, 사용자가 지정한 Codex 작업 로그 경로나 동등한 로컬 작업 로그 경로를 사용한다.
- 파일명 형식은 `YYYYMMDD_VRBeatSaber_Log.md`를 사용한다.
- 날짜가 바뀌면 새 파일을 생성한다.
- 외부 작업 로그에는 현재 프로젝트와 관련된 내용만 기록한다.
- 외부 작업 로그 파일은 백업하지 않는다.
- 기존 날짜 전용 통합 로그가 있고 프로젝트별 로그로 분리되어 있다면, 통합 로그는 삭제해도 된다.

## 백업 정책

- 코드, 씬, 프리팹, 데이터 파일을 수정하기 전에는 백업을 생성한다.
- 단, `Log.md`, `Task.md`, `AGENTS.md`, 외부 작업일 로그는 백업하지 않는다.
- 씬 백업은 `Assets/Scenes/Backup` 폴더에 저장한다.
- 일반 코드/데이터 백업은 `Backup/` 하위 적절한 폴더에 저장한다.
- 사용자가 명시적으로 백업하지 말라고 한 파일은 백업하지 않는다.

## Unity 작업 규칙

- Unity 관련 작업은 가능하면 Unity MCP / Editor 상태를 확인하며 진행한다.
- 스크립트 수정 후에는 Unity 컴파일 상태를 확인한다.
- 주요 씬/런타임 변경 후에는 가능한 범위에서 Play Mode, 콘솔 로그, 캡처 등으로 검증한다.
- Quest 3S 실기 테스트가 필요한 항목은 에디터 검증과 실기 검증을 구분해서 `Task.md`에 기록한다.
- 실기 테스트가 불가능한 상황에서는 에디터에서 확인 가능한 범위만 진행하고, 실기 항목은 대기로 남긴다.

## Git / GitHub 관리

- 원격 저장소는 `https://github.com/ssa25879/VRRhythmGame.git`이다.
- 기본 브랜치는 `main`이다.
- Unity 재생성 폴더와 로컬 상태 파일은 Git에 포함하지 않는다.
  - `Library/`
  - `Temp/`
  - `Logs/`
  - `UserSettings/`
  - `.vs/`
  - `.vscode/`
  - `.codex/`
  - `.claude/`
  - `Backup/`
  - `Assets/Screenshots/`
- Git 작업 전에는 `git status`를 확인한다.
- 커밋 전에는 불필요한 파일이나 대용량 재생성 파일이 포함되지 않았는지 확인한다.
- 작업을 완료하면 필요한 경우 커밋하고 원격 `main`에 push한다.
